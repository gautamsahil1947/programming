answer to question no1 part (A)

On click , onsubmit and onreset differ on the basis of the triggers
onsubmit executes when the submit buttion is pressed. on reset triggers when the reset button is pressed
on click triggers when some option is selected like ok or cancel in the alertsor when
certain options from a list when selected either block or open other inputs(for eg when clicked on non-veg, veg foods disappear )

it can be used for username and passwords for the login pages as follows :
  1. Onsubmit events trigger when the submit button is pressed.
  2. Onreset events occur when the reset button is pressed and this triggers the js logic which refreshes the form
  3. onclick events are triggered when confirming passwords and when some options are selected like the login via other methods like gmail etc.
  4. onclick is also used in capchas for validation of the user
